2642. 设计可以求最短路径的图类

题目

给你一个有 n 个节点的 有向带权 图，节点编号为 0 到 n - 1 。图中的初始边用数组 edges 表示，其中 edges[i] = [fromi, toi, edgeCosti] 表示从 fromi 到 toi 有一条代价为 edgeCosti 的边。

请你实现一个 Graph 类：

  • Graph(int n, int[][] edges) 初始化图有 n 个节点，并输入初始边。
  • addEdge(int[] edge) 向边集中添加一条边，其中 edge = [from, to, edgeCost] 。数据保证添加这条边之前对应的两个节点之间没有有向边。
  • int shortestPath(int node1, int node2) 返回从节点 node1 到 node2 的路径 最小 代价。如果路径不存在，返回 -1 。一条路径的代价是路径中所有边代价之和。

示例 1：

提示：

  • 1 <= n <= 100
  • 0 <= edges.length <= n * (n - 1)
  • edges[i].length == edge.length == 3
  • 0 <= fromi, toi, from, to, node1, node2 <= n - 1
  • 1 <= edgeCosti, edgeCost <= 106
  • 图中任何时候都不会有重边和自环。
  • 调用 addEdge 至多 100 次。
  • 调用 shortestPath 至多 100 次。

解题思路

题目要求实现 Graph 类，一共有三个类方法，一个构造方法，一个添加边的方法，一个求最短路径的方法。前两个属于简单问题，没有什么难度，重点在于求有向加权图的最短路径的那一个方法。

之前数据结构课设中，我选择的课设题目便是解决有向加权图的最短路径的求取方法，所以刚好用上了。

  • Floyd算法

    • 算法思想：是一种基于动态规划的多源最短路算法。
    • 优点：代码非常容易理解，而且很优美。是一种在具有正或负（但没有负环）加权图中找到最短路径的算法，容易理解，可以算出任意两个节点之间的最短距离，代码编写简单
    • 缺点：时间复杂度比较高，不适合计算大量数据 时间复杂度:O(n^3)；
    • 代码实现

      • public void shortestEdges(){
        minEdges = edges;

        for(int i = 0; i < n; i++)
            for(int j = 0; j < n; j++){
                if(minEdges[i][j] == 0) minEdges[i][j] = Integer.MAX_VALUE;
                if(i == j) minEdges[i][j] = 0;
            }

        for(int k = 0; k < n; k++)
            for(int i = 0; i < n; i ++)
                if(minEdges[i][k] != Integer.MAX_VALUE)
                    for(int j = 0; j < n; j++)
                        if(minEdges[k][j] != Integer.MAX_VALUE && minEdges[i][j] > minEdges[i][k] + minEdges[k][j])
                            minEdges[i][j] = minEdges[i][k] + minEdges[k][j];
        
    }

  • SPFA算法

    • 算法介绍

      • 最短路径快速算法（Shortest Path Faster Algorithm (SPFA)），一般认为是队列优化的Bellman-Ford 算法，是一个用于求解有向带权图单源最短路径的算法。
      • 这一算法被认为在随机的稀疏图上表现出色，并且适用于带有负边权的图对于无向图，可通过将每个无向边视作两条有向边以采用 SPFA 算法。
      • SPFA的复杂度大约是O(kE),k是每个点的平均进队次数(一般的，k是一个常数，在稀疏图中小于2)
      • 其优于狄杰斯特拉算法的方面是边的权值可以为负数、实现简单，缺点是时间复杂度过高，高达 O(VE)。

    • 算法思想

      • 用数组dis记录每个结点的最短路径估计值，用邻接表或邻接矩阵来存储图G。
      • 采取动态逼近法：设立一个先进先出的队列用来保存待优化的结点，优化时每次取出队首结点u，并且用u点当前的最短路径估计值对离开u点所指向的结点v进行松弛操作，如果v点的最短路径估计值有所调整，且v点不在当前的队列中，就将v点放入队尾。
      • 这样不断从队列中取出结点来进行松弛操作，直至队列空为止

    • 代码实现

      • public static int spfa() {
        Queue<Integer> queue = new LinkedList<>();
        Arrays.fill(dist, INF);
        dist[1] = 0;
        queue.offer(1);
        st[1] = true;

        while (!queue.isEmpty()) {
            int t = queue.poll();
            st[t] = false;//从队列中取出来之后该节点st被标记为false,代表之后该节点如果发生更新可再次入队

            for (int i = h[t]; i != -1; i = ne[i]) {
                int j = e[i];
                if (dist[j] > dist[t] + w[i]) {
                    dist[j] = dist[t] + w[i];
                    if (!st[j]) {//当前已经加入队列的结点，无需再次加入队列，即便发生了更新也只用更新数值即可，重复添加降低效率
                        queue.offer(j);
                        st[j] = true;
                    }
                }
            }
        }
        return dist[n];
}
